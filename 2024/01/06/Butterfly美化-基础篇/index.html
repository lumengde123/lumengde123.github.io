前言

【说明】为了更好的总结归纳优化博客的各个功能，方便查阅和使用，目录的顺序与_config.yml内配置的顺序有一些不同，但是基本保持有序。使用时多利用Ctrl+F查阅想要的功能或者在主题配置中跳转到相应位置。阅读完这篇文章，将会对_config.butterfly.yml有比较全面的了解

网站个人资料修改

打开博客根目录的配置文件_config.yml，在这里面可以修改网站基本资料，例如标题、副标题和语言等。

参数：

  • title：网站主题，对应导航栏左侧和主页顶部图的标题
  • subtitle：副标题
  • description：对网站内容的简要概括，会显示在搜索引擎结果中
  • keywords：网站关键字，有助于搜索引擎识别和索引网站内容
  • author：会改变侧边用户栏作者的名字以及每篇文章的默认作者
  • language：网站使用的语言，默认为en（英文），该主题还支持zh-CN (简体中文)和zh-TW (繁体中文)
  • timezone：网站使用的时区，中国的时区为’Asia/Shanghai’

以下开始，基本都是在_config.butterfly.yml中进行配置，复制关键词在文件中Ctrl + F可以快速找到对应位置

头像

顶部图

如果不想显示顶部图，可直接配置 disable_top_img: true

顶部图的获取顺序（若都没有配置，则默认显示主题色）

页面顶部图的获取顺序：各自配置的 top_img > 配置文件的 default_top_img

文章页顶部图的获取顺序：各自配置的 top_img > cover > 配置文件的 default_top_img

以下是各个配置及其所代表的设置，可以根据需要进行配置

配置 详解
index_img 主页的 top_img
default_top_img 默认的 top_img，当页面的 top_img 未配置时显示这个
archive_img 归档页面的默认 top_img
tag_img tag 子页面的默认 top_img
tag_per_img tag 子页面的 top_img，可分别单独配置每个 tag 的 top_img（不推荐）
category_img category 子页面 的 默认 top_img
category_per_img category 子页面的 top_img，可分别单独配置每个 category 的 top_img（不推荐）

不推荐为每个 tag 和每个 category 都配置不同的顶部图，因为配置太多会拖慢生成速度
标签总页/分类总页/自建页面 和 各个文章页 的 top_img ，可以在对应的 md 页面的front-matter中，添入top_img进行设置

以上配置可以使用的值：

  • false：关闭该功能
  • 空值：显示默认的 top_img，否则显示默认顔色主题色（文章页top_img留空的话，会显示 cover 的值）
  • img链接：显示外部图片链接
  • transparent：透明效果
  • 颜色值：
    • HEX值【#0000FF】
    • RGB值【rgb(0,0,255)】
    • 顔色单词【orange】
    • 渐变色【linear-gradient( 135deg, #E2B0FF 10%, #9F44D3 100%)】

文章封面的获取顺序：Front-matter 的 cover > 配置文件的 default_cover > false

最新评论
在侧边栏显示最新评论板块(3.1.0起支持)；最新评论只会在刷新时才会去读取，并不会实时变化

由于 API 有 访问次数限制，为了避免调用太多，主题默认存取期限为 10 分钟。也就是説，调用后资料会存在 localStorage 里，10分钟内刷新网站只会去 localStorage 读取资料。 10 分钟期限一过，刷新页面时才会去调取 API 读取新的数据。（ 3.6.0 新增了 storage 配置，可自行配置缓存时间）

文章页相关配置

页面meta显示

这个选项用来在文章标题下方显示文章的相关信息。不论在主页还是详情文章页都能显示。

主页文章节选

因为主题UI的关係，主页文章节选只支持自动节选和文章页description。
在butterfly里，有四种可供选择:

  • description： 只显示description
  • both： 优先选择description，如果没有配置description，则显示自动节选的内容
  • auto_excerpt：只显示自动节选
  • false：不显示文章内容

页面锚点

开启页面锚点后，当你在进行滚动时，页面链接会根据当前标题ID进行替换

文章复制和版权声明

1.复制文章内容时后面自动加上版权信息

2.为文章展示版权和许可协议

由于Hexo 4.1开始，默认对网址进行解码，以至于如果是中文网址，会被解码，可设置decode: true来显示中文网址。

如果有文章（例如：转载文章）不需要显示版权，可以在文章Front-matter单独设置：copyright: false

从3.0.0开始，支持对单独文章设置版权信息，可以在文章Front-matter单独设置

Toc目录

在文章页，会有一个目录，用于显示TOC。

为特定的文章配置
在文章md文件的Front-matter，加入toc_number和toc配置，设置值为true或者false即可。

主题会优先以文章Markdown的Front-matter配置为准，如果没有，才以主题配置文件中的配置为准

打赏按钮

在文章结尾设置打赏按钮，可以放上自己的收款二维码。
图片需要自己加，放在/public/img/。当然也可以自己设置位置。
参数link如果设置了链接，点击图片会跳转到相应的打赏链接

文章编辑按钮

在文章标题旁边显示一个编辑按钮，点击会跳转到对应的链接去。

该功能允许读者或作者通过网站直接访问到文章在版本控制系统（如 GitHub）中的编辑页面
用处(来自chatGPT)：

  • 便于编辑，及时更新或修正文章
  • 便多人共同编辑和修改，促进读者参与贡献和改进文档的积极性
  • 提高内容的透明度，读者可以了解文章的源头和编辑历史，这有助于建立信任和可靠性

相关文章

当文章封面设置为 false 时，或者没有获取到封面配置，相关文章背景将会显示主题色。
相关文章推荐的原理是根据文章tags的比重来推荐

文章分页按钮

当文章封面设置为 false 时，或者没有获取到封面配置，分页背景将会显示主题色。
可设置分页的逻辑，感觉打开比较好

文章过期提醒

可设置是否显示文章过期提醒，以更新时间为基准
