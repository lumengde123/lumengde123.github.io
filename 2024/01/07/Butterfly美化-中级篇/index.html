引入自定义css和js

在魔改博客的时候，可能需要自定义css和js
在博客根目录 source/ 下新建css文件夹，在里面新建一个 custom.css 文件，
在博客根目录 source/ 下新建一个 js 文件，在里面新建一个 custom.css 文件。
不建议放在theme/butterfly文件夹里面，因为更新主题的时候会被覆盖丢失
可以在source/下新建个config文件夹，把css、js、img都放里面,方便管理

修改主题配置文件 _config.butterfly.yml，将它们引入

只要配置路径写对了，hexo g的时候它也会自动创建没有的文件

主页设置

主页副标题

打开主题配置文件，修改相应配置

参数：

  • enable：开启副标题
  • effect：打字效果，如果关闭只会显示sub第一行字
  • source：副标题调用第三方服务，且优先级高于sub
  • sub：多个自定义副标题循环显示
    source数值 作用
    1 调用一言网的一句话
    2 调用一句网语录
    3 调用今日诗词
    false 关闭调用

参数及解释：

参数 备选值/类型 释义
priority number 【可选】过滤器优先级，数值越小，执行越早，默认为10，选填
enable true/false 【必选】控制开关
enable_page path/all 【可选】填写想要应用的页面的相对路径（即路由地址）,如根目录就填’/‘,分类页面就填’/categories/‘。若要应用于所有页面，就填’all’，默认为all
timemode date/updated 【可选】时间显示，date为显示创建日期，updated为显示更新日期,默认为date
layout.type id/class 【可选】挂载容器类型，填写id或class，不填则默认为id
layout.name text 【必选】挂载容器名称
layout.index 0和正整数 【可选】前提是layout.type为class，因为同一页面可能有多个class，此项用来确认究竟排在第几个顺位
default_descr text 默认文章描述
swiper_css url 【可选】自定义的swiper依赖项css链接
swiper_js url 【可选】自定义的swiper依赖项加js链接
custom_css url 【可选】适配主题样式补丁
custom_js url 【可选】swiper初始化方法

主页置顶 文章

方法1是Hexo自带的，butterfly直接就能用。方法2适用于不支持sticky属性的主题，并且还有隐藏文章功能

方法1：
Hexo默认提供了 hexo-generator-index 插件，支持置顶文章（PR #51 版本后），在文章配置中添加 sticky 属性即可。
使用方法：直接在文章的 Front-matter 里添加 sticky: true | 数字 属性来把这篇文章置顶。数值越大，置顶的优先级越大。
不足：不支持隐藏文章，且大多数主题渲染置顶标签的方式是读取文章中的 top 属性而不是sticky属性。

这里补充一个可以使用hide: true隐藏文章的方法：Hexo的Butterfly下如何隐藏部分文章不在首页显示

方法2：
hexo-generator-index-custom插件支持置顶和隐藏文章，并且sticky和top属性都可用，该插件基于官方插件所以_config.yml配置也无需修改

首先卸载默认的首页生成器插件hexo-generator-index
