Hexo-Github搭建个人博客

发表于2024-01-07|更新于2024-01-07
|阅读量:

前言

官方文档：Hexo

一些参考教程：
1.静态网站搭建笔记 GitHub Page+Hexo - 知乎 (zhihu.com)
2.【2023最新版】Hexo+github搭建个人博客并绑定个人域名_hexo部署github绑定自定义域名-CSDN博客

Hexo框架搭建博客网站

流程概括：安装Git、安装Node.js、安装Hexo并部署本地仓库。

使用 Hexo 构建静态网站并将其托管在 Git 版本控制系统上，利用 Node.js 运行 Hexo 以便在本地进行调试和预览。

安装Git

Git 可以让你在开发中管理和追踪代码版本，方便团队协作和代码的备份。

打开Git - Downloading Package (git-scm.com)，选择**64-bit Git for Windows Setup.**

基本无脑下一步就行。在Choosing the default editor used by Git这一步选择Use Visual Studio Code as Git’s default editor。

安装完可以在cmd中输入git --version指令检查是否安装成功

安装Node.js

Node.js 是 Hexo 的运行环境，安装 Node.js 后可以在你的计算机上运行 Hexo。

打开Node.js (nodejs.org)，下载左边那个LTS版本

一样无脑安装。在cmd中输入检查是否安装成功，返回版本信息。

安装Hexo并部署本地仓库

Hexo 是一个静态网站生成器，可以帮助你用 Markdown 编写博客内容，然后生成静态网页。

首先配置 npm（Node 包管理器）的镜像源为淘宝 NPM 镜像，目的是提升下载速度，因为国外网站访问很慢。当然，有魔法可以略过这一步。

安装hexo命令行工具（-g 表示全局安装，意味着安装的包将在系统的整个环境中都可用，可以在cmd中直接调用相关的命令。）

进入自定义的文件路径，初始化hexo。

安装项目所需的依赖：npm 会读取项目中的 package.json 文件，并安装该项目所依赖的所有包。

配置 Git 全局用户信息

当你提交代码时，Git会使用这个用户名作为代码提交的作者名称。这个邮箱地址将用于关联你的提交记录。

这样就创建好了一个本地的博客，用的是Hexo的默认主题
执行以下指令就可以在本地看见自己的博客了

本地博客部署到Github上

流程：创建Github的库以供上传、生成SSH密钥方便验证身份、部署本地博客到Github上
这一步略麻烦，遇到出错要耐心，然后要确保网络通畅(确保魔法好使，能连上Github)

创建Github Pages远程仓库

注册github账号并登录，进入个人github主页

点击上方右侧+号，New Repository，新建一个仓库名和自己 Github 账号 ID 相同的仓库并创建

例：若Github账号为lumegnde123，则新建仓库名为lumengde123.github.io

生成SSH密钥

使用 SSH 协议与 GitHub 进行通信，通过更加安全和便捷的方式来验证你的身份

在cmd中输入以下指令，多个回车直到出现代码表情。
意思是使用 RSA 加密算法生成SSH密钥，并将你的 GitHub 注册邮箱作为注释添加到生成的密钥中。

打开C:\Users\Administrator\.ssh，系统在这里生成了一对 SSH 密钥：本机私人密钥 id_rsa 和公共密钥 id_rsa.pub

公钥 可以添加到你的 GitHub 或其他 Git 托管服务账户上，以便你能够使用 SSH 协议与服务器进行安全的通信和操作。
私钥 则留存在你的计算机上，用于与公钥配对进行身份验证。

这里打开 id_rsa.pub 文件，复制里面所有内容。
在Github中，点击头像->Settings->SSH and GPG keys->New SSH key，title随意填，下面粘贴ssh公钥。

当后面使用hexo d部署博客时，将会用SSH 协议与 GitHub 进行通信，GitHub 将会根据你计算机提交的私钥与你账户中存储的公钥进行匹配。只有在密钥匹配成功的情况下，你才能够顺利地进行诸如推送代码、拉取仓库等 Git 操作，而无需每次都输入密码。

输入指令检查连接有没问题

这里会卡住，意思是访问Github超时。需要换个网络(切换魔法)，当然仅供参考，不管他也行，后面部署博客的时候再慢慢试hexo d指令。

博客部署到Github上

【这一部分比较麻烦，容易遇到坑，多上网查解决办法】
打开博客根目录的_config.yml文件，分别修改url和deploy的内容

注意你Github上创建的账号.github.io仓库默认是main分支，需要改成master(不会改百度哦~)；或者这里改成branch: main。这一步是让hexo知道把文件推到Github仓库中的哪个分支去。

安装hexo-deploy-git插件，这样就可以使用Hexo的部署命令

执行三件套指令，以后每次部署都需要用上

当执行hexo deploy时，Hexo 会将public目录中的文件和目录推送至_config.yml中指定的远端仓库和分支中，并且完全覆盖该分支下的已有内容

第一次hexo d会问个问题，必须填yes，然后按回车慢慢等

【补充】本地修改文章常用hexo g && hexo d，改好了再用hexo d部署到Github上，每次部署后Github上都会记录部署时间、提交者、部署内容等。

【注意】第一次部署要等很久，还容易出现这个报错。

这是网络问题导致的。我查了一整天没整明白，突然就行了
就是多换节点死命试。每次都耐心等待，某次意外就成了

每换一个节点就ping github.com试一下，不超时的才能用，超时的这里同样会卡住报错

当出现INFO Deploy done: git提示时，代表部署成功。第一次成功后，后续更新都不会卡了。

这样本地hexo博客就同步到Github上了

在浏览器输入：你的账号.github.io就能访问默认博客首页

自定义域名

如果你希望输入你自己的域名自动跳转到github博客首页，可以先去阿里云购买域名，实名认证，然后将该域名和github域名进行绑定。

我这里简单说一下流程，不够详细。

1.在阿里云购买域名，完成实名认证。
2.打开阿里的域名控制台->云解析DNS->某个域名-解析设置->添加记录，添加两条
3.记录类型：CNAME；主机记录：一个是www，一个是@

打开blog文件夹，在resource文件夹下新建文件，重命名为CNAME（没有后缀）

打开CNAME，填入你购买的并设置好解析的域名

cmd进入blog文件夹，输入指令hexo clean && hexo g && hexo d

然后在浏览器中访问你的域名，可以跳转到自己的github博客网站

如果你不希望一篇文章（post/page）使用主题处理，请在它的 front-matter 中设置 layout: false
