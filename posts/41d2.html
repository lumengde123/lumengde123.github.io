主页置顶文章

方法1是Hexo自带的，butterfly直接就能用。方法2适用于不支持sticky属性的主题，并且还有隐藏文章功能。

方法1

Hexo默认提供了 hexo-generator-index 插件，支持置顶文章，在文章配置中添加 sticky 属性即可。

使用方法：直接在文章的 Front-matter 里添加 sticky: true 或 数字 属性来把这篇文章置顶。数值越大，置顶的优先级越大。
不足：不支持隐藏文章，且大多数主题渲染置顶标签的方式是读取文章中的 top 属性而不是sticky属性。

这里补充一个在方法1下可以使用hide: true隐藏文章的方法：Hexo的Butterfly下如何隐藏部分文章不在首页显示

方法2：
hexo-generator-index-custom插件支持置顶和隐藏文章，并且sticky和top属性都可用，该插件基于官方插件所以_config.yml配置也无需修改

10 分钟期限一过，刷新页面时才会去调取 API 读取新的数据。（ 3.6.0 新增了 storage 配置，可自行配置缓存时间）

网站个人资料修改

打开博客根目录的配置文件_config.yml，在这里面可以修改网站基本资料，例如标题、副标题和语言等。

参数：

  • title：网站主题，对应导航栏左侧和主页顶部图的标题
  • subtitle：副标题
  • description：对网站内容的简要概括，会显示在搜索引擎结果中
  • keywords：网站关键字，有助于搜索引擎识别和索引网站内容
  • author：会改变侧边用户栏作者的名字以及每篇文章的默认作者
  • language：网站使用的语言，默认为en（英文），该主题还支持zh-CN (简体中文)和zh-TW (繁体中文)
  • timezone：网站使用的时区，中国的时区为’Asia/Shanghai’

以下开始，基本都是在_config.butterfly.yml中进行配置，复制关键词在文件中Ctrl + F可以快速找到对应位置

头像

顶部图

如果不想显示顶部图，可直接配置 disable_top_img: true

顶部图的获取顺序（若都没有配置，则默认显示主题色）

页面顶部图的获取顺序：各自配置的 top_img > 配置文件的 default_top_img

文章页顶部图的获取顺序：各自配置的 top_img > cover > 配置文件的 default_top_img

以下是各个配置及其所代表的设置，可以根据需要进行配置

配置 详解
index_img 主页的 top_img
default_top_img 默认的 top_img，当页面的 top_img 未配置时显示这个
archive_img 归档页面的默认 top_img
tag_img tag 子页面的默认 top_img
tag_per_img tag 子页面的 top_img，可分别单独配置每个 tag 的 top_img（不推荐）
category_img category 子页面 的 默认 top_img
category_per_img category 子页面的 top_img，可分别单独配置每个 category 的 top_img（不推荐）

不推荐为每个 tag 和每个 category 都配置不同的顶部图，因为配置太多会拖慢生成速度
标签总页/分类总页/自建页面 和 各个文章页 的 top_img ，可以在对应的 md 页面的front-matter中，添入top_img进行设置

以上配置可以使用的值：

  • false：关闭该功能
  • 空值：显示默认的 top_img，否则显示默认顔色主题色（文章页top_img留空的话，会显示 cover 的值）
  • img链接：显示外部图片链接
  • transparent：透明效果
  • 颜色值：
    • HEX值【#0000FF】
    • RGB值【rgb(0,0,255)】
    • 顔色单词【orange】
    • 渐变色【linear-gradient( 135deg, #E2B0FF 10%, #9F44D3 100%)】
