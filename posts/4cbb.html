Hexo框架搭建博客网站

流程概括：安装Git、安装Node.js、安装Hexo并部署本地仓库。

使用 Hexo 静态网站生成工具来构建静态网站并将其托管在 Git 版本控制系统上，利用 Node.js 运行 Hexo 以便在本地进行调试和预览。

  • 安装Git

Git 可以让你在开发中管理和追踪代码版本，方便团队协作和代码的备份。

打开Git - Downloading Package (git-scm.com)，选择**64-bit Git for Windows Setup.**

基本无脑下一步就行。在Choosing the default editor used by Git这一步选择Use Visual Studio Code as Git’s default editor。

安装完可以在cmd中输入git --version指令检查是否安装成功

  • 安装Node.js

Node.js 是 Hexo 的运行环境，安装 Node.js 后可以在你的计算机上运行 Hexo。

打开Node.js (nodejs.org)，下载左边那个LTS版本

一样无脑安装。在cmd中输入检查是否安装成功，返回版本信息。

  • 安装Hexo并部署本地仓库

Hexo 是一个静态网站生成器，可以帮助你用 Markdown 编写博客内容，然后生成静态网页。

首先配置 npm（Node 包管理器）的镜像源为淘宝 NPM 镜像，目的是提升下载速度，因为国外网站访问很慢。当然，有魔法可以略过这一步。

安装hexo命令行工具（-g 表示全局安装，意味着安装的包将在系统的整个环境中都可用，可以在cmd中直接调用相关的命令。）

进入自定义的文件路径，初始化hexo。

安装项目所需的依赖：npm 会读取项目中的 package.json 文件，并安装该项目所依赖的所有包。

配置 Git 全局用户信息

当你提交代码时，Git会使用这个用户名作为代码提交的作者名称。这个邮箱地址将用于关联你的提交记录。

这样就创建好了一个本地的博客，用的是Hexo的默认主题，后面会安装美化主题。
执行以下指令就可以在本地看见自己的博客了

本地博客部署到Github上

流程：创建Github的库以供上传、生成SSH密钥方便验证身份、部署本地博客到Github上
这一步略麻烦，遇到出错要耐心，然后要确保网络通畅(确保魔法好使，能连上Github)

  • 创建Github Pages远程仓库

注册github账号并登录，进入个人github主页

点击上方右侧+号，New Repository，新建一个仓库名和自己 Github 账号 ID 相同的仓库并创建

例：若Github账号为lumegnde123，则新建仓库名为lumengde123.github.io

  • 生成SSH密钥

使用 SSH 协议与 GitHub 进行通信，通过更加安全和便捷的方式来验证你的身份。

在cmd中输入以下指令，多个回车直到出现代码表情。
意思是使用 RSA 加密算法生成SSH密钥，并将你的 GitHub 注册邮箱作为注释添加到生成的密钥中。

打开C:\Users\Administrator\.ssh，系统在这里生成了一对 SSH 密钥：本机私人密钥 id_rsa 和公共密钥 id_rsa.pub

公钥 可以添加到你的 GitHub 或其他 Git 托管服务账户上，以便你能够使用 SSH 协议与服务器进行安全的通信和操作。
私钥 则留存在你的计算机上，用于与公钥配对进行身份验证。

这里打开 id_rsa.pub 文件，复制里面所有内容。

当后面使用hexo d部署博客时，将会用SSH 协议与 GitHub 进行通信，GitHub 将会根据你计算机提交的私钥与你账户中存储的公钥进行匹配。只有在密钥匹配成功的情况下，你才能够顺利地进行诸如推送代码、拉取仓库等 Git 操作，好处是无需每次都输入密码。

使用技巧

  • 本地修改文章，如果只改变文字的话，保存笔记并刷新网页就能看到改变内容；
  • 如果改变了网页效果，先ctrl+C退出，然后hexo g && hexo s重新生成(渲染)网页和启动本地服务。
  • 博客要部署到Github上，需要hexo clean && hexo g && hexo d，先删除上次部署的缓存文件，重新生成网页再部署上去。
  • 每次部署后Github的Deployments都会记录部署时间、提交者、部署内容等。

自定义域名

xxx.github.io这个域名不太好看。所以我们可以将其绑定到我们自己买的域名上。

如果你希望输入你自己的域名自动跳转到github博客首页，可以先去阿里云购买域名，实名认证，然后将该域名和github域名进行绑定。

我简单说一下流程，有不懂的可以百度解决：

  • 在阿里云购买域名，完成实名认证。
  • 打开阿里的域名控制台->云解析DNS->选择你的域名-解析设置，添加一条记录
  • 打开cmd命令窗口，输入nslookup 你的github昵称.github.io，可以得到你的域名对应的DNS的IP。前四个为IPV6地址，Github暂不支持，用来解析会出错（实测）。最后四个是IPV4地址，我们用来域名解析。

在你的域名-解析设置中添加1~4个记录都可

  • 在本地项目的source/下面新建个文件，命名：CNAME，填你的域名，www.xxxx.xx。然后hexo d更新到github。

    如果DNS check报错，先确定你的域名解析填对了。也可以试试这个：github pages，自定义域名开启https遇到Domain does not resolve to the GitHub Pages server. For more information，记得看它评论区提示。

    然后等DNS的缓存刷新（可能要十几分钟，一定要耐心等待，这玩意更新就是很慢），再刷新看看应该就绿了。

然后在浏览器中输入你的域名，此时就可以跳转到你的github博客网站了（但是速度一样慢）

本节参考教程：[为 Github Pages 绑定域名 - cometeme的个人博客](https://cometeme.github.io/web/2018/08/为GitHubPages绑定域名.html#:~:text=使用自己的域名来访问博客 1 1. 修改 DNS 解析的数据 在设置 Github,Using a custom domain with GitHub Pages。)

butterfly主题安装

butterfly界面很…原始，但是可以根据个人爱好自定义。

cmd进入博客项目根目录，下载主题。

安装pug 和 stylus 渲染器。
Pug（以前的Jade）用于简化HTML模板的编写，而Stylus则用于更简洁、更灵活地书写CSS样式。

修改项目根目录下的_config.yml文件（称为站点配置文件），开启主题。

为了减少升级主题带来的不便，可以复制一份themes/butterfly/中 config.yml的内容到根目录的_config.landscape.yml里，并重命名为 _config.butterfly.yml。

Hexo会自动合并主题中的config.yml和 _config.butterfly.yml ，如果存在同名配置，会优先使用config.butterfly.yml的配置。

所以像和博客网址相关联的固定资料可以设置在_config.yml中，比如博客的标题、作者信息和邮箱等等资料;而和主题样式相关的配置放在 _config.butterfly.yml 中，方便以后更换主题。

butterfly美化教程可以在标签的butterfly中查阅（页脚的相关推荐应该有）

重装博客

  • 重装网站上的博客

  • 重装本地博客
    提前备份好了本地博客文件，重新初始化一个仓库并将备份文件的文章、页面、图片、代码依次放入并调整一下即可。

  • 从远程仓库恢复本地仓库
    如果换了电脑，想从github仓库的博客文件恢复本地博客并同步：

    • 先把博客文件从远程仓库下载下来并解压。
    • 按照本文顺序重新创建博客。
    • 将解压的文件覆盖到对应文件夹下。
    • 根据原博客效果，重新安装插件和编写配置配置信息。这个需要自己慢慢调了。

然后浏览器打开看看本地能否正常使用
